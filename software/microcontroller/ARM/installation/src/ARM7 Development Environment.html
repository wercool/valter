Setup

GCC Toolchain

Go to www.gnuarm.com/files and download the source code files for

  • GCC-4.1.1.tar.bz2
  • binutils-2.17.tar.bz2

navigate to the download directory 

cd ~/Downloads

binutils

You must compile and install binutils prior to compiling GCC 

tar -jxvf binutils-2.17.tar.bz2
cd binutils-2.17
./configure --disable-werror --target=arm-elf --prefix=/usr/local/arm7
sudo make all install

Make this path available for use: 

export PATH=$PATH:/usr/local/arm7/bin

Add to ~/.bashrc file so that it is always avalible

GCC

tar -jxvf gcc-4.1.1.tar.bz2
cd gcc-4.1.1
./configure --target=arm-elf --prefix=/usr/local/arm7 --enable-languages=c --with-newlib
su
make all-gcc install-gcc
exit

arm-elf-gcc --version

If you are using a Olimex AT91SAM7-H256 header board for the AT91SAM7S-256 micro-controller download the Olimex supplied Blink LED Source. This program is written for the Olimex AT91SAM7-P256 development board so you may want to edit the code to blink a certain pin or the green LED on the header board.

Using the following commands you can compile a simple C source code into a .elf executable and then use arm-elf-objcopy to convert the .elf file into a binary file that can be programming to the device using SAM-BA. 

arm-elf-gcc -mcpu=arm7tdmi -g -c -o main.elf main.c
arm-elf-objcopy main.elf -O binary main.bin

You should see something like this: 

brent@brent-VB:~/ARM/Blinking_LED$ pwd

brent@brent-VB:~/ARM/Blinking_LED$ arm-elf-objcopy main.elf -O binary main.bin
brent@brent-VB:~/ARM/Blinking_LED$ ls

using Minimalistic Blink Program

This worked like a charm! Minimal Blink For Sam7S64 and Sam7S256 Thank you Wolfgang Wieser (triplespark SAM7 for making your great tutorial and simplistic files! 

Change symlink AT91SAM7.h from AT91SAM7S64.h to AT91SAM7S256.h if using the Olimex -H256

Edit the ld_flash.cmd file from its S64 version text to: 

/* Specify the memory areas for AT91SAM7S256: */
MEMORY 
{
	flash	: ORIGIN = 0,          LENGTH = 256K		/* FLASH EPROM		*/	
	ram		: ORIGIN = 0x00200000, LENGTH = 64K  	/* static RAM area	*/
}

/* Define a global symbol _stack_end (see analysis in annotation above): */
_stack_end = 0x203FFC;    /* AT91SAM7S256 */

To Compile: crt.s is a low-level initialization assembly file from ATMEL This MUST be included into the binary loaded onto the chip or the chip won't start up properly! 

arm-elf-as -mapcs-32 crt.s -o crt.o

Compiling main.c (Blink): 

arm-elf-gcc -I. -c -fno-common -O2 -g main.c

We now need to link our two .o object files (crt.o and main.o) so that both the start-up code and main blinking code run. The resulting file will be named main.elf 

arm-elf-ld -T ld_flash.cmd -nostartfiles -o main.elf crt.o main.o

Lastly, we must create a binary file (SAM-BA required, although other programming methods ask for HEX) to program onto the AT91SAM7S256 

arm-elf-objcopy --output-target=binary main.elf main.bin



Writing a Blinking LED Program

This guide is aimed to help relieve some of the troubles associated with learning embedded ARM development. I will take you through the creation of each file to help cover what, why, and how for the minimum files needed to use the ATMEL SAM7S series devices. Note that many other examples available online as well as files available from ATMEL contain much of the same details and "standard" file names. I hope to walk you through each step an provide a much more thorough base. I am learning much of this as I write this code and figure it out as well and hopefully I can spare you the same level of pain that is sometimes associated!

crt.s - Assembly Start-up File

The ARM SAM7S256 needs to be "booted" with a start up sequence, starting the necessary processes and entering into the "main" C function. A good ARM assembly reference found online is locate here, of course it is also recommended to view more formal and supported documents from ATMEL, ARM, and texts.

Necessary SAM7 References: 

AT91SAM7S Series Preliminary (Memory Map & PIO+other maps)
ATMEL application note [6131A]

The ATMEL application 6131A states that Initialization code located at address 0 after reset must: 

- Mark the default entry point for the initialization code
- Set up the exception vectors
- Initialize the memory system
- Initialize any critical I/O devices
- Initialize any register required by the interrupt system
- Enable interrupts (if handled by the initialization code)
- Change the processor mode if necessary 
- Change Processor state if necessary

Creating A Makefile

The best approach to compiling a program is to create a Makefile. A Make file automates the compiling process to reduce the chance of a user error when issuing commands to compile the make files and dependencies associated with the program. This tutorial will be written for the Linux ARM7 toolchain. It is encouraged to practice making a simple makefile with a simple c program (i.e. helloworld) on your Linux computer before writing one for ARM7.

Future: Explain all actions, flags, and other attributes to the make file 

#################################
#	AT91SAM7S256 MakeFile	#
#	Brenton Salmi, KB1LQD	#
#	December 22, 2010	#
#################################

# Variables (GNU-ARM)
CC	= arm-elf-gcc		# C Compiler
LD	= arm-elf-ld		# Linker
AR	= arm-elf-ar		#
AS	= arm-elf-as		# Assembler
CP	= arm-elf-objcopy	# .elf converter (HEX or BIN)

# GNU ARM Flags (Options)
CFLAGS	= -I./ -c -fno-common -O0 -g
AFLAGS	= -ahls -mapcs-32 -o crt.o
LFLAGS	= -Map main.map -Tdemo_at91sam7_blink_flash.cmd
CPFLAGS	= --output-target=binary


OBJECTS = crt.o main.o lowlevelinit.o

#make clean command
clean:
	-rm $(OBJECTS) crt.lst main.lst main.out main.bin main.hex main.map main.dmp

all: main.out
	$(CP) $(CPFLAGS) main.out main.bin

main.out: $(OBJECTS) demo_at91sam7_blink_flash.cmd
	$(LD) $(LFLAGS) -o main.out $(OBJECTS)

crt.o: crt.s
	$(AS) $(AFLAGS) crt.s > crt.lst

main.o: main.c
	$(CC) $(CFLAGS) main.c

lowlevelinit.o: lowlevelinit.c
	$(CC) $(CFLAGS) lowlevelinit.c

Using SAM-BA to load Blinking LED with SAM7S-H256

After much frustration, a good working BIN file and course code can be downloaded from the Olimex website so that a solid reference can be used to learn from. Using the ATMEL SAM-BA USB boot-loader program. Download the Blinking LED EW-ARM zip file containing both the source code and compiled .bin file. Unzip this folder and connect to the Olimex SAM7S-H256 board. Browse to the "Flash" tab in the lower portion of the screen. Select "Enable Flash Access" and click execute, BE CAREFUL NOT TO SELECT THE SECURITY BIT OPTION. Next, select "Erase All Flash" and click Execute. It is now time to load on the .bin file that is supplied withing the unzipped folder, select it in the send file box and make sure the first memory address it is programming is 0x0100000. Send the file and it should do so successfully, use the button in the program to check if the code on the device matches that supplied in the .bin. The code should now execute if the reset button is pressed or USB power cycled.

Controlling Pins Using PIO

Programming the SAM7 device is much different than programming other computers or embedded devices. The Parallel Input/Output registers allow the programming to control the pins and assign or read functions to them.
