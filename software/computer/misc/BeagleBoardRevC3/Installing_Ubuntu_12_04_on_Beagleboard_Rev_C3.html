Tutorial: Installing Ubuntu 12.04 on Beagleboard Rev. C3

Leave a reply

(This will hopefully be the first of many tutorial posts on various subjects. This will primarily be an aid for my bad memory; hopefully others will find this useful as well.)

Recently I was tasked by one of my professor to set-up Ubuntu 12.04 on a Beagleboard Rev. C3. I have taken a look at the Beagleboard in the past, but have never owned a Beagleboard.

The materials one would need are:

  • Beagleboard Rev. C3: I am being specific here because I have only used these steps on this version. These steps should apply to other non-xM Beagleboard, and maybe the Beaglebones.
  • 8 GB or larger SD card: Just the installation of Ubuntu 12.04 will take up 3GB, so 16GB or 32GB would not be a bad idea.
  • Serial to USB converter with the DB9 connector.
  • SD card editor, or computer with SD card editor.

Here are the steps I used to set-up the Beagleboard.

  • Proceed to this webpage, ARM/OMAP Desktop install. This page describes the installation process for both Ubuntu 12.10 and 12.04 on OMAP4 and OMAP3. Follow the instruction for Ubuntu 12.04 until reaching the section “Booting the image”.
  • At this point, pull the SD card out of the reader and insert it back in. If the instructions were followed correctly, there should be two partitions:
    • A small FAT32 partition holding the boot directory.
    • A larger Ext3 partition, 3~4GB, holding the file system.

Several files in the file system require modification before the system is ready to boot. Please keep in mind, that these alterations are to be done on the SD card’s partitions as root; be extremely careful.

  • Replace the boot.scr currently in the boot partition. First create a new file “bootcmd” with the following content 
    fatload mmc 0:1 0x80000000 uImage
fatload mmc 0:1 0x81600000 uInitrd
setenv bootargs 'vram=12M omapfb.mode=dvi:1280x720MR-16@60 mpurate=${mpurate} console=ttyO2,115200n8 root=/dev/mmcblk0p2 fixrtc'
bootm 0x80000000 0x81600000

    Compile it into the “boot.scr” file using command

    mkimage -A arm -O linux -T script -C none -a 0 -e 0 -n "Beagleboard boot script" -d bootcmd boot.scr

    Copy the new “boot.scr” into the boot partition. This new “boot.scr” will enable the on-board serial port during linux boot; otherwise only the monitor will be displaying content.

  • Create a new file in the Ext3 partition’s /etc/init/ dir, “serial.conf”, with the following content: 
    start on stopped rc RUNLEVEL=[2345]
stop on runlevel [!2345]

respawn
exec /sbin/getty 115200 ttyO2

    This will enable the command line login over serial.

Now create a new user for this system. The Ubuntu desktop distribution does not have a default user account to log into, and the root account has a random password; the user account is normally created during the Ubuntu set-up process, which requires a monitor, mouse, and keyboard. The steps I am outlining is meant for setting up the system over serial only. Again, these alterations are to be done on the SD card’s partitions as root; be extremely careful. Navigate to /etc/, and modify these files as root:

  • passwd: create a user.
  • group: create group for the user.
  • shadow: insert the hash of the password for the user; use mkpasswd to create a SHA-512 password hash.
  • sudoers: give the user sudoer rights. This is a bit tricky because the default permission is read-only for owner and group. So first, do the following as root to give write permission to root 
    chmod 640 sudoers

    now modify the file. Now run these command as root to change the permission back to read only.

    chmod 440 sudoers
chown root:root sudoers

    If the permission and ownership for sudoers is not right, sudo will not run.

  • Create the home folder for the user as root, and than change the ownership: 
    mkdir /home/(user name)
chown (user name):(user group) /home/(user name)

Unmount the two partitions, and insert the SD card into the Beagleboard. Power on the Beagleboard while holding down the User1 button (one closest to the edge). This forces the chip to load from the SD card instead of from on-board NAND. During the boot process, X-boot will load, U-boot will load, and the Kernel will be copied into RAM and started.

On first boot, the kernel will not mount the root file immediately; it will run a partition editor to expand the 3~4GB Ext3 partition to take up the entire SD card. Once a command prompt is presented, login with the user account created earlier.

With the system booting up, the next stage is to connect the Beagleboard to the internet. I choose the TP-LINK TL-WN772N USB Wifi adaptor; this adaptor uses a chipset that is known to have great support in Linux. Since the Ubuntu installed is a desktop distribution, it will contain all the drivers an x86 distribution would have. Plug the adaptor into the Beagleboard and check that the correct driver was installed using lsusb. Next:

  • As root, create /etc/wpa_supplicant.conf and add the following content 
    ctrl_interface=/var/run/wpa_supplicant
eapol_version=1
ap_scan=1

network={
    scan_ssid=1
    ssid="non-secure-net-0"
    key_mgmt=NONE
}

network={
    scan_ssid=1
    ssid="secure-net-1"
    proto=WPA RSN
    key_mgmt=WPA-PSK
    pairwise=CCMP TKIP
    psk="password"
}
  • As root, append the following to /etc/network/interfaces 
    # Wireless
auto wlan0
iface wlan0 inet dhcp
pre-up rfkill unblock all
pre-up wpa_supplicant -B -Dwext -iwlan0 -c/etc/wpa_supplicant.conf
post-down killall -q wpa_supplicant
  • As root, run 
    ifdown wlan0
ifup wlan0

    The system will connect to one of the wireless network listed and perform DHCP.

With that, Ubuntu 12.04 is installed on the Beagleboard Rev. C3 and connected to the internet.

UPDATE: The current procedure for booting the Beagleboard requires holding down User1 button. The solution was to force clear the on board NAND. During the current bootup sequence, stop at the Uboot stage; when Uboot starts and prompts for any key press to stop the auto-boot, press something. Now, U-boot will present a command line interface. Do the following to whip the entire NAND chip.

OMAP3 beagleboard.org #
OMAP3 beagleboard.org # nand erase.chip

After completing the NAND clear, power cycle the board. The Beagleboard should boot from SD card with out using User1 button.
